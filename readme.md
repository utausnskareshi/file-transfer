# File Transfer - iPhone ⇔ PC

同一 Wi-Fi ネットワーク上の iPhone と Windows PC の間で、ファイルを双方向に送受信するアプリケーションです。
iPhone 側は **Safari ブラウザのみ**使用し、専用アプリのインストールは不要です。

---

## 動作環境

| 項目 | 内容 |
|---|---|
| PC 側 OS | Windows 10 / 11 |
| 開発環境 | Visual Studio 2022 以降（.NET 8.0） |
| ターゲットフレームワーク | .NET 8.0 Windows（WinForms） |
| iPhone 側 | Safari ブラウザ（iOS 15 以降推奨） |
| ネットワーク | iPhone と PC が同一 Wi-Fi に接続されていること |

## プロジェクト構成

```
(リポジトリルート)
├── FileTransfer.slnx              ソリューションファイル（XML 形式）
├── .gitignore                     Git 除外設定
├── readme.md                      このファイル
└── FileTransfer\
    ├── FileTransfer.csproj        プロジェクトファイル（.NET 8 WinForms）
    ├── Program.cs                 エントリーポイント
    ├── MainForm.cs                メインウィンドウ ロジック
    ├── MainForm.Designer.cs       メインウィンドウ コントロール定義
    ├── WebServer.cs               HTTP サーバー本体（API 実装）
    └── HtmlContent.cs             iPhone 向け Web UI（HTML/CSS/JS）
```

### 各ファイルの役割

#### `Program.cs`
アプリケーションのエントリーポイント。`MainForm` を起動します。

#### `MainForm.cs` / `MainForm.Designer.cs`
WinForms のメインウィンドウです。以下を提供します。
- 共有フォルダの選択（フォルダブラウザダイアログ）
- ポート番号の設定（デフォルト: 8080）
- サーバーの開始 / 停止
- iPhone からアクセスする URL の表示とクリップボードコピー
- ファイル転送ログのリアルタイム表示
- 初回起動時の URL ACL 登録・ファイアウォールルール追加（UAC 経由）

#### `WebServer.cs`
`System.Net.HttpListener` を使った軽量 HTTP サーバーです。
iPhone からの HTTP リクエストを受け付け、以下の API を提供します。

| メソッド | パス | 内容 |
|---|---|---|
| GET | `/` | iPhone 向け Web UI（HTML）を返す |
| GET | `/api/files` | 共有フォルダのファイル一覧を JSON で返す |
| GET | `/api/download/{ファイル名}` | 指定ファイルをダウンロードさせる |
| POST | `/api/upload` | `multipart/form-data` でファイルを受信・保存する |
| POST | `/api/delete` | 指定ファイルを削除する |

#### `HtmlContent.cs`
iPhone の Safari で表示する Web ページを C# の文字列定数として保持します。
HTML / CSS / JavaScript がすべて含まれており、外部ファイルは不要です。
Apple のデザインガイドラインに沿ったモバイルファーストの UI になっています。

## セットアップ手順

### 1. Visual Studio でビルドする

1. `FileTransfer.slnx` を Visual Studio で開く
2. メニュー → `ビルド` → `ソリューションのビルド`（または `Ctrl+Shift+B`）
3. エラーがなければビルド完了

> **注意**: ソリューションファイルの拡張子は `.slnx`（XML 形式）です。
> .NET SDK 9 以降で採用された新形式で、Visual Studio 2022 17.12 以降および Visual Studio 2026 で対応しています。

### 2. PC と iPhone を同じ Wi-Fi に接続する

- PC：Wi-Fi またはイーサネットでルーターに接続
- iPhone：**同じ** Wi-Fi ネットワークに接続

### 3. アプリケーションを起動する

1. Visual Studio で `F5` を押して実行（またはビルドした `exe` をダブルクリック）
2. メインウィンドウが表示される

## 使い方

### PC 側の操作

```
┌─────────────────────────────────────────────────┐
│ 設定                                             │
│  共有フォルダ: [C:\Users\...\FileTransfer] [...] [開く] │
│  ポート:       [8080]  [サーバー開始]            │
│  接続URL:      [http://192.168.1.x:8080/] [コピー] │
└─────────────────────────────────────────────────┘
│ 転送ログ                                         │
│  [09:00:00] サーバー起動: http://192.168.1.10:8080/ │
│  [09:00:00] リッスン中: http://+:8080/（全 NIC 対象）│
│  [09:00:01] リクエスト受付ループを開始しました。  │
│  [09:01:15] 接続: 192.168.1.5:54321 → GET /     │
│  [09:01:20] アップロード: IMG_0001.jpg (3.2 MB)  │
└─────────────────────────────────────────────────┘
```

1. **共有フォルダを選択する**
   - `...` ボタンを押してフォルダを選択する
   - デフォルトは `マイドキュメント\FileTransfer`
   - `開く` ボタンでエクスプローラーからフォルダを確認できる

2. **ポート番号を設定する**（通常はデフォルトの `8080` のまま使用）

3. **「サーバー開始」を押す**
   - 初回のみ UAC ダイアログが 1〜2 回表示されます（URL ACL 登録・ファイアウォール設定）
   - 「接続URL」欄に iPhone からアクセスする URL が表示される
   - 例: `http://192.168.1.10:8080/`
   - ログ欄に「リクエスト受付ループを開始しました」と表示されれば準備完了

4. **URL を iPhone に送る**
   - `コピー` ボタンで URL をクリップボードにコピー
   - メール・メモ・AirDrop 等で iPhone に送信する

### iPhone 側の操作

1. **Safari で URL を開く**
   - PC の「接続URL」に表示されているアドレスを Safari のアドレスバーに入力する
   - 例: `http://192.168.1.10:8080/`
   - PC のログ欄に「接続: 192.168.1.x:xxxxx → GET /」と表示されれば接続成功

2. **iPhone → PC へファイルを送る（アップロード）**
   - 画面上部の「ファイルを送信」エリアをタップ
   - ファイルの選択元が表示される：
     - **「フォトライブラリ」** → カメラロールから写真・動画を選択（複数可）
     - **「写真を撮る」** → その場で撮影してアップロード
     - **「ファイルを選択」** → ファイルアプリから選択
   - 選択後、「アップロード」ボタンをタップ
   - プログレスバーで進捗を確認できる

3. **PC → iPhone へファイルを受け取る（ダウンロード）**
   - 画面下部の「共有ファイル」一覧に PC の共有フォルダのファイルが表示される
   - ↓ ボタンをタップしてダウンロード
   - ダウンロードしたファイルは iPhone の「ファイル」アプリ → ダウンロードに保存される
   - 写真は長押し → 「"写真"に追加」でカメラロールにも保存できる

4. **ファイルを削除する**
   - ✕ ボタンをタップすると確認ダイアログの後に削除される（PC の共有フォルダから削除）

## 初回起動時の権限設定

サーバー開始時に以下の処理が自動で行われます。**いずれも初回のみ**です。

### 1. URL ACL 登録

```
ポート 8080 でのリッスンに管理者権限が必要です。
自動で URL 予約を追加しますか？
```

- **「はい」を選択する**と UAC ダイアログが表示されます
- 承認すると `netsh http add urlacl` コマンドが実行され、ポートの使用権限が登録されます
- **「いいえ」を選択した場合**は、Visual Studio または exe を「管理者として実行」で起動してください

### 2. ファイアウォールルール追加

URL ACL 登録の直後、別の UAC ダイアログが表示されます。

- 承認すると `netsh advfirewall firewall add rule` コマンドが実行されます
- ルール名: `FileTransfer_Port8080`（ポート番号に応じて変化）
- これにより iPhone などの外部デバイスからの接続が許可されます
- **ファイアウォールルールを追加しないと iPhone から接続できません**

### 手動でコマンドを実行する場合

管理者権限のコマンドプロンプトで以下を実行します（8080 は使用ポートに合わせて変更）。

```cmd
rem URL ACL 登録
netsh http add urlacl url=http://+:8080/ user=DOMAIN\UserName

rem ファイアウォールルール追加
netsh advfirewall firewall add rule name="FileTransfer_Port8080" dir=in action=allow protocol=TCP localport=8080
```

## セキュリティについて

- **ファイル名のサニタイズ**: ダウンロード・削除 API ではディレクトリトラバーサル攻撃（`../` 等）を防ぐためにファイル名を検証しています
- **アクセス範囲**: サーバーは同一ネットワーク上のすべてのデバイスからアクセスできます。**信頼できるネットワーク（自宅 Wi-Fi 等）でのみ使用してください**
- **認証なし**: ユーザー認証は実装していません。パスワードによる保護が必要な場合は `WebServer.cs` の `HandleRequest` メソッドに認証処理を追加してください

## カスタマイズ

### ポートのデフォルト値を変更する

`MainForm.Designer.cs` の以下の行を変更します。

```csharp
nudPort.Value = 8080;  // ← 任意のポート番号に変更
```

### 共有フォルダのデフォルトパスを変更する

`MainForm.cs` の `MainForm_Load` メソッドを変更します。

```csharp
var defaultFolder = Path.Combine(
    Environment.GetFolderPath(Environment.SpecialFolder.MyDocuments),
    "FileTransfer");
```

### Web UI のデザインを変更する

`HtmlContent.cs` の `GetPage()` メソッド内の HTML / CSS / JavaScript を編集します。
CSS 変数（`:root` 内の `--primary` 等）を変更すると配色をまとめて調整できます。

### 対応 MIME タイプを追加する

`WebServer.cs` の `GetMimeType` メソッドにケースを追加します。

```csharp
".xxx" => "application/xxx",
```

## トラブルシューティング

### iPhone から接続できない

1. iPhone と PC が**同じ Wi-Fi** に接続されているか確認する
2. PC のログ欄に「ファイアウォールルール 'FileTransfer_Port8080' を追加しました」または「既に登録済みです」と表示されているか確認する
   - 表示されていない場合は上記「手動でコマンドを実行する場合」を参照して手動追加する
3. 「接続URL」に表示されている IP アドレスが Wi-Fi の IP か確認する（複数の NIC がある環境では意図しない NIC の IP が表示される場合がある）
4. PC のログ欄に「接続: ...」のログが出ているか確認する
   - 出ていない場合はファイアウォールや URL ACL の設定を見直す
   - 出ている場合はアプリ側の処理エラーの可能性があるため、ログのエラーメッセージを確認する

### サーバー起動時に「アクセスが拒否されました」エラーが出る

`HttpListenerException: アクセスが拒否されました` が発生する場合、URL ACL が未登録です。

- 「URL 予約を追加しますか？」ダイアログで「はい」を選択し、UAC を承認する
- または管理者として Visual Studio / exe を実行する

### ファイルのアップロードが途中で止まる

- 大容量ファイル（1 GB 以上）の場合、`WebServer.cs` の `HandleUpload` がリクエストボディ全体をメモリに読み込むため、メモリ不足になる可能性があります
- ストリーミング処理への変更が必要です（`request.InputStream.CopyToAsync(ms)` をストリーム逐次処理に変更）

### ファイル名が文字化けする

- `WebServer.cs` の `ExtractFileName` が RFC 5987 形式（`filename*=UTF-8''...`）と従来形式（`filename=...`）の両方に対応しているため、通常は発生しません
- 問題が発生した場合はログに表示されるファイル名を確認してください
